Nora — Claude Instructions

System reference for Claude Code scheduled sessions and integrations

System Overview

Nora is a voice AI PM agent for LimeLight Marketing. She joins Zoom meetings via Recall.ai, listens to conversation through real-time transcription, responds when triggered by wake word ("Nora"), and speaks back using ElevenLabs TTS. She has persistent memory and a task queue that external agents (like Claude Code) can process.

Architecture

  • Server: Node.js/Express on Railway
  • Voice pipeline: Zoom → Recall.ai transcription webhook → Wake word detection → Claude Haiku → ElevenLabs TTS → Recall.ai output audio → Zoom
  • AI: Claude Haiku 4.5 for voice responses, Claude Haiku 4.5 for memory/task extraction
  • Storage: JSON files on Railway Volume (/data/) — memory and tasks persist across deploys
  • Dashboard: Web UI at root URL for meeting joins, task management, memory management

Wake Word & Triggers

  • Voice: Say "Hey Nora" or "Nora" followed by a question in a Zoom meeting
  • Chat: Type @nora your question in Zoom chat
  • Stop phrases: "Stop Nora", "Nora stop", "hold on", "never mind" — cuts off mid-speech

Memory System

Nora has persistent long-term memory stored as a JSON array. Each memory is a fact string with metadata. Memory is appended to the system prompt on every Claude call so Nora always has access to everything she's been told.

Auto-extraction: After each voice response, a background Haiku call checks if someone explicitly asked Nora to remember something. Only explicit "remember this" requests and action items with clear owners/deadlines are saved.

Storage: /data/nora-memory.json on Railway Volume (falls back to nora-memory.json locally)

Task Queue

Nora's task queue is the bridge between meetings and action. When someone asks Nora to do something specific in a meeting (schedule a meeting, send an email, create a Teamwork task), it gets extracted and added to the queue as a pending task.

For Claude Code scheduled sessions: Fetch pending tasks, execute them using MCP tools (Teamwork, Slack, Gmail, Google Calendar), then mark each one complete by ID. The complete endpoint is idempotent — calling it twice won't double-do anything.

Storage: /data/nora-tasks.json on Railway Volume

Task API

GET /tasks
List all tasks. Filter with ?status=pending or ?status=done
[
  {
    "id": "nora-1710432000000-a3f2",
    "action": "Schedule follow-up meeting with Kyle",
    "detail": "Discuss Q2 campaign results",
    "assignee": "Kyle Tapper",
    "due": "next Tuesday",
    "source_channel": "slack:C0123ABCDEF",
    "source_user": "U0123ABCDEF",
    "status": "pending",
    "created": "2026-03-14T18:00:00.000Z",
    "completed": null
  }
]
POST /tasks
Add a task manually
// Request body:
{
  "action": "Send weekly report to Brandee",
  "detail": "Include Teamwork metrics",
  "assignee": "Brandee Johnson",
  "due": "Monday 8am"
}

// Response: { "ok": true, "id": "nora-1710432000000-a3f2" }
PATCH /tasks/:id/complete
Mark a task as done (idempotent — safe to call multiple times)
PATCH /tasks/nora-1710432000000-a3f2/complete

// Response: { "ok": true, "task": { ... } }
// If already done: { "ok": true, "already": true, "task": { ... } }
DELETE /tasks/:id
Delete a task permanently
DELETE /tasks/nora-1710432000000-a3f2
// Response: { "ok": true }

Claude Code Scheduled Session Workflow

This is the recommended workflow for a scheduled Claude Code session that processes Nora's task queue:

1. Fetch pending tasks:
   GET {BASE_URL}/tasks?status=pending

2. For each pending task, determine the right action:
   - "Schedule a meeting..." → use Google Calendar MCP to create event
   - "Send an email to..." → use Gmail MCP to draft/send
   - "Create a task in Teamwork..." → use Teamwork MCP to create task
   - "Send a Slack message..." → use Slack MCP to post message
   - "Remind [person] about..." → determine best channel and notify

3. Execute the action using the appropriate MCP tool

4. Notify the requester that it's done (posts as Nora):
   POST {BASE_URL}/notify
   {
     "channel": "C0123ABCDEF",   // from task.source_channel (strip "slack:" prefix)
     "user": "U0123ABCDEF",      // from task.source_user (used for DMs)
     "text": "Done — scheduled the follow-up with Kyle for Tuesday at 2pm."
   }
   Use "channel" for channel replies, "user" for DMs.
   If source_channel starts with "slack:", strip the prefix to get the channel ID.
   If source_channel is "zoom", skip Slack notify or DM the source_user instead.

5. Mark the task as done:
   PATCH {BASE_URL}/tasks/{task_id}/complete

6. Optionally, add a memory about what was done:
   POST {BASE_URL}/memory
   { "fact": "Sent Q2 report to Brandee on 2026-03-14", "source": "auto" }

Available MCP Integrations

  • Teamwork (twprojects): Create/update tasks, milestones, projects, time logs
  • Teamwork Desk (twdesk): Manage support tickets, customers, messages
  • Google Calendar (gcal): Create/update events, find free time, check availability
  • Gmail: Search messages, read threads, create drafts
  • Slack: Send messages, search channels, read threads
